Bi.Core.Repository.get,
Bi.Core.Repository.synchronize

var repository = new Bi.Core.Repository();

// Using one string argument
repository.get('Module1').queue(function (result) {
    // Do something with result.Module1 ...
});

// Using one array argument
repository.get(['Module1', 'Module2']).queue(function (result) {
    // Do something with result.Module1 and result.Module2 ...
});

// Using more arguments...
repository.get('Module1', 'Module2').queue(function (result) {
    // Do something with result.Module1 and result.Module2 ...
});

Récupère les données de configuration de différents modules, quel que soit le mode de navigation (online/offline).

Cette méthode se base sur Bi.database.run.mode() pour déterminer si les données sont à récupérer sur le serveur distant ou en local. 

repository.synchronize().queue(function (success) {
    if (success) {
        // The repository has been synchronized for offline navigation...
    }
});

Après synchronisation, les données de configuration sont stockées dans window.localStorage pour une utilisation offline.
