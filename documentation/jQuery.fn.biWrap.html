jQuery.fn.biWrap

Interface utilisateur (UI) de l'application Bi.

<div class="bi-wrap bi-font">
    <!-- Consulter la démo pour connaître le balisage attendu par le plugin -->
</div>

Les classes .bi-wrap et associées (.bi-top, .bi-middle, ...), définissent l'apparence de l'interface.

La classe optionnelle .bi-font définie la police de caractères propre à l'application Bi. Si l'application Bi est inclus en tant que bloc dans la page HTML d'une autre application (comme le CRM par exemple) alors il peut être intéressant de ne pas ajouter cette classe pour profiter de la police de caractères de l'application globale.

$(selector).biWrap({
	hide: {
		top: false,
		bottom: false,
		sidebar: false,
		head: false
	},
	hideTrigger: {
		top: false,
		bottom: false,
		sidebar: false
	},
	fixedMain: false
});
Options
Nom Valeurs Description
hide object Définie les régions de l'interface à ne pas afficher.
hideTrigger object Définie les régions de l'interface pour lesquelles le bouton afficher/masquer n'est pas disponible.
fixedMain false (default), true Détermine si le conteneur principal reste "fixe" même lorsque la panneau latéral est ouvert.
Dans ce cas, le panneau latéral s'ouvre en recouvrant("overlay") le contenu principal.
Méthodes
Nom Paramètres Retour Description
show layout, trigger jQuery instance Si trigger n'est pas renseigné ou vaut false alors la méthode affiche la région correspondante au layout. Si trigger vaut true alors la méthode affiche le bouton afficher/masquer associé à la région.
hide layout, trigger jQuery instance Idem que la méthode show mais cette fois pour masquer la région ou le bouton associé au layout.
toggle layout, trigger jQuery instance Idem que les méthodes hide et show de manière alternée.
fixedMain boolean jQuery instance Lorsque le conteneur principal est fixe, le panneau latéral s'ouvre en le recouvrant.
Sinon, la largeur du conteneur principal est réduite lorsque le panneau latéral est ouvert.
isFixedMain boolean Retourne true si le conteneur principal est fixe et false sinon.
